In-class Exercise: More JavaScript

Part 1: if and else

  1. Start a blank Glitch site in your account.
  2. In script.js, we want to add an if statement. Delete anything in the file. Then add this to the file: 
    if (4 === 5) {
  console.log('4 is 5');
}
  3. Confirm that your function works by viewing the resulting site (Show Live) and looking at the console in developer tools. There should be nothing in the console!
  4. Let's add an else statement. Delete the existing code in script.js and add this: 
    if (4 === 5) {
  console.log('4 is 5');
}
else {
  console.log('4 is not 5');
}
  5. Look in the console now and you should see "4 is not 5".

Part 3: Filtering data using the NYC Open Data API

  1. Remix this Glitch site in your account or continue working from the site from the previous example.
  2. Open the JavaScript file.
  3. Change the borough filter to QUEENS and confirm that the map data is updated.

Part 6: Listening for Leaflet events

  1. Remix this Glitch project into your Glitch account.
  2. In the JavaScript, listen for more map events and log out the name of the event. For example, copy 
    map.on('moveend', function () {
  console.log('moveend');
});
    paste those lines (so the moveend listener still exists) and change moveend in the new listener, like this: 
    map.on('click', function () {
  console.log('click');
});
    Don't forget to change what is logged out, too, so you can see which event happened.
  3. Do the above process until you have five different map events. Pay special attention to:
    • click
    • dblclick
    • mousedown
    • mouseup
    • move
    How do the above events differ from each other?
