In-class Exercise: GitHub

Part 1: Put a site on GitHub Pages

In this exercise, we'll put a Glitch site online with GitHub Pages. This process is long but we outline the steps here and you can refer to tonight's slides for images showing the process.

  1. Open GitHub and log in if you need to.
  2. Click on the + at the top right of the page, then New Repository.
  3. Give the repository a name and check Initialize this repository with a README.
  4. Under the repository's Settings, scroll down to GitHub Pages select master under Source.
  5. Open any Glitch site in your account.
  6. Click on the project name at the top left of the page.
  7. Click Advanced Options.
  8. Click Export to GitHub. You may have to give Glitch access first.
  9. In the dialog that opens, type your GitHub username, a / and the GitHub repository name from earlier. For example, my GitHub username is ebrelsford. If my repository is leaflet-map-class10, I would type ebrelsford/leaflet-map-class10.
  10. Go back to GitHub and open the repository that you created earlier. Click Branch, then glitch.
  11. Click New Pull Request.
  12. Click Create pull request.
  13. On the next page, click Merge pull request.
