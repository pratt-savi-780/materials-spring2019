In-class Exercise: JavaScript

Part 1: Look at some JavaScript

  1. With a partner, open this Glitch site. You do not have to remix this site yet, but do click the Show live button to see what it does.
  2. Look at the HTML and CSS for the site. What looks familiar? What looks unfamilar? What's your best guess for what the unfamiliar parts do?
  3. Look at the JavaScript for the site (script.js). What is the code doing? Do you notice any patterns in the code?

Part 2: Change some settings on a Leaflet map

  1. Remix this Glitch site in your account.
  2. In the script.js file on line 4 there should be a line like this one: 
    var map = L.map('map').setView([34.03, -82.20], 5);
    On this line:
    1. 34.03 is the center latitude,
    2. -82.20 is the center longitude, and
    3. 5 is the zoom level.
  3. Try editing the zoom and see what changes on the map.
  4. Try editing the latitude and longitude. If you want to be more precise, try using bboxfinder to find a helpful center. (Keep an eye on the order of the latitude and longitude though!)

Part 3: Change the basemap

  1. Continuing with your code from the previous part, let's change the basemap. On line 7 there is a url like this: 
    https://stamen-tiles.a.ssl.fastly.net/toner/{z}/{x}/{y}.png
  2. Check out the OpenStreetMap wiki page on tiles, some of which are free to use.
  3. Let's try the layer called "OSM no labels". One of the URLs listed there looks like this: 
    https://tiles.wmflabs.org/osm-no-labels/${z}/${x}/${y}.png
    We need to remove the $s so it looks like this: 
    https://tiles.wmflabs.org/osm-no-labels/{z}/{x}/{y}.png
    Then we can add replace the URL on line 7 with that and our map should include a different base map.
  4. Pick another layer from the wiki page to replace the above URL with. As another example, this hillshade basemap: 
    http://c.tiles.wmflabs.org/hillshading/${z}/${x}/${y}.png
    Delete the old URL in your code and paste the new URL in, making sure you remove the dollar signs ($): 
    http://c.tiles.wmflabs.org/hillshading/{z}/{x}/{y}.png

Part 4: Load your own data

  1. Remix this Glitch project into your Glitch account.
  2. Go to gist.github.com and find the map data you created last week in class. If you can't find the data or didn't create any, make some simple data now with geojson.io.
  3. Download the data from Gist by clicking the Raw button and saving the resulting page.
  4. Upload the data to your Glitch assets folder (as you would with an image).
  5. Copy the URL for the uploaded data and paste it in line 12 of the JavaScript, replacing the URL that is currently there.
  6. View the resulting page and confirm that your data is loaded on the map.
