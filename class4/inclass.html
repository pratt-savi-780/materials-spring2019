In-class Exercise: JavaScript

Part 1: Working with objects

  1. Remix this Glitch site in your account.
  2. Note that the school name (SCHOOLNAME) is on the developer tools console.
  3. Edit the script.js file to also log the school type (SCH_TYPE) on the console.
  4. Add another variable to script.js, named row2. This should be an object like the row object already in the file. The values in the object should be like this:
    • BORO: 'K'
    • BORONUM: 2
    • SCHOOLNAME: 'P.S. 101 THE VERRAZANO'
    • SCH_TYPE: 'Elementary'
    • ADDRESS: '2360 BENSON AVENUE'
  5. In script.js, log some of the values from this object to the console.
  6. In the console, try accessing the values of the object interactively by typing: 
    row2.
    and seeing the members of the object listed for you.

Part 2: Change some options on a Leaflet map

  1. Remix this Glitch site in your account.
  2. We would like to set options on the map that change the way the map works. Options are often stored in objects, so let's create an object called mapOptions before line 4: 
    var mapOptions = {
};
  3. In mapOptions we'll set some values. Let's start with setting dragging to false. Update mapOptions to 
    var mapOptions = {
  dragging: false
};
  4. We'll set more options, but first give these to Leaflet when it creates the map. Change the line 
    var map = L.map('map').setView([38.500893,-98.745117], 5);
    to 
    var map = L.map('map', mapOptions).setView([38.500893,-98.745117], 5);
  5. The Leaflet documentation contains the many options available when creating a map. Try at least two more options and feel free to get rid of the dragging option we set above. For example, you might set attributionControl or zoomControl to false.

Part 3: Creating functions

  1. Remix this Glitch site in your account or continue working from the site from the previous example.
  2. Create a function at the bottom of the JavaScript file by copying this into the file: 
    function zoomToPratt() {
  map.setView([40.6919, -73.9649], 18);
}
  3. The resulting page should be exactly as before—the function doesn't do anything until you call it. Open developer tools, go to the console tab, and type 
    zoomToPratt()
    When you press [enter] the map should zoom to the Pratt Brooklyn campus.
  4. Create another function in your JavaScript file after zoomToPratt that zooms somewhere else in the world.

Part 4: Load a GeoJSON file from NYC Open Data

  1. Go to New York's Open Data portal.
  2. Search for school districts.
  3. Open the first result, called School districts.
  4. Click Export.
  5. Download the data as a GeoJSON.
  6. Remix this Glitch project into your Glitch account.
  7. Upload the GeoJSON to the assets folder and load the data on the map.
