In-class Exercise: Events

Part 1: Listening for Leaflet events

  1. Remix this Glitch project into your Glitch account.
  2. In the JavaScript, listen for more map events and log out the name of the event. For example, copy 
    map.on('moveend', function () {
  console.log('moveend');
});
    paste those lines (so the moveend listener still exists) and change moveend in the new listener, like this: 
    map.on('click', function () {
  console.log('click');
});
    Don't forget to change what is logged out, too, so you can see which event happened.
  3. Do the above process until you have five different map events. Pay special attention to:
    • click
    • dblclick
    • mousedown
    • mouseup
    • move
    How do the above events differ from each other?

Part 2: Leaflet layer events

  1. Remix this Glitch project into your Glitch account.
  2. Note that there is an event listener for clicks on features on the map. View the site and confirm that this works as expected.
  3. Look at the events available for interactive layers and add simple event listeners for two more. I recommend mouseover and mouseout.

Part 3: Text input events

  1. Remix this Glitch project into your Glitch account.
  2. The text input currently does not work—all the data is always loaded. In the loadData function, change url to make it load data where complaint_type is equal to the input text. Currently url is defined as: 
    var url = "https://data.cityofnewyork.us/resource/fhrw-4uyv.geojson";
    and we want to add a parameter to the end to make it search for specific complaint_types: 
    var url = "https://data.cityofnewyork.us/resource/fhrw-4uyv.geojson?complaint_type=" + text;
  3. Verify this works by searching for specific complaint_types. Rodent and Illegal Parking should both work.

Part 4: like

  1. Continue with the Glitch project from the previous part.
  2. We want to make the search input more flexible using like. For example this will look like this: 
    $where=lower(complaint_type) like '%rodent%'
    Currently your url looks like this: 
    var url = "https://data.cityofnewyork.us/resource/fhrw-4uyv.geojson?complaint_type=" + text;
    Replace it with this: 
    var url = "https://data.cityofnewyork.us/resource/fhrw-4uyv.geojson$where=" + encodeURIComponent("lower(complaint_type) like '%rodent%'");
  3. Confirm that the above gets data where complaint_type contains rodent.
  4. Make the url more flexible—instead of always looking for complaint_types containing rodent, replace rodent with the contents of the text parameter passed to the loadData function.

Part 5: forEach

  1. Create a new Glitch project that is a webpage.
  2. In the JavaScript file, create an array of numbers. For example: 
    var allTheNumbers = [1, 2, 3, 4.1, 10000000];
  3. Use forEach to log out each value in the array. For example: 
    allTheNumbers.forEach(function (number) {
  // Do something with number here
});

Part 6: Checkbox events

  1. Remix this Glitch project into your Glitch account.
  2. Add checkboxes for the other boroughs. You should only have to change the HTML to make this work.
  3. If you have time, change the HTML and loadData to filter by some of the values in complaint_type.
